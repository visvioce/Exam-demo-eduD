## 三、分层职责

### 1. Controller 层
- 接收 HTTP 请求
- 参数校验
- 调用 Service
- 返回统一响应

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    @Autowired
    private UserService userService;
    
    @GetMapping("/{id}")
    public Result<UserResponse> getById(@PathVariable Long id) {
        User user = userService.getById(id);
        return Result.success(user);
    }
}
```

### 2. Service 层
- 业务逻辑处理
- 事务管理
- 调用 Mapper

```java
@Service
public class UserService extends ServiceImpl<UserMapper, User> {
    
    public User login(String username, String password) {
        User user = lambdaQuery().eq(User::getUsername, username).one();
        // 验证密码...
        return user;
    }
}
```

### 3. Entity 层
- 对应数据库表
- 使用注解映射

```java
@Data
@TableName("users")
public class User {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String username;
    private String password;
    private String nickname;
    private String role;
    private String status;
    private LocalDateTime createdAt;
    private Integer deleted;
}
```

### 4. Mapper 层
- 继承 BaseMapper
- 复杂 SQL 写 XML

```java
@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 简单 CRUD 已自带
    // 复杂查询在此定义
    List<User> selectByRole(@Param("role") String role);
}
```

### 统一响应
```java
@Data
public class Result<T> {
    private Integer code;
    private String message;
    private T data;
    
    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMessage("操作成功");
        result.setData(data);
        return result;
    }
    
    public static <T> Result<T> error(String message) {
        Result<T> result = new Result<>();
        result.setCode(400);
        result.setMessage(message);
        return result;
    }
}
```

## 六、开发规范

### 命名规范
- 包名：全小写，如 `com.southcollege.exam`
- 类名：大写驼峰，如 `UserController`
- 方法名：小写驼峰，如 `getUserById`
- 常量：全大写下划线，如 `MAX_PAGE_SIZE`

### 代码风格
- 使用 Lombok 简化 getter/setter
- 使用 MyBatis-Plus 的 lambda 查询
- 统一返回 Result 对象
- 异常统一处理

### Git 提交规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式
refactor: 重构
perf: 性能优化
test: 测试相关
chore: 构建/工具相关
```

## 八、注意事项

1. **JSON 字段**：使用 `@TableField(typeHandler = JacksonTypeHandler.class)` 注解
2. **API Key 加密**：AI 配置的 API Key 使用 AES 加密存储
3. **权限控制**：使用 Spring Security + JWT 实现 RBAC
4. **数据校验**：使用 `@Valid` 和 `@NotNull` 等注解
5. **事务管理**：Service 层方法使用 `@Transactional`
