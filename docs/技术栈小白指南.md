## 一、项目整体架构

### 1.1 什么是前后端分离？

**通俗理解**：
- 想象一家餐厅
- **前端** = 服务员 + 餐厅装修（顾客看到的）
- **后端** = 厨房 + 仓库（顾客看不到的）
- **前后端分离** = 服务员和厨师分开工作，通过"点菜单"（API接口）交流

**为什么要分离？**
1. **分工明确**：前端负责好看，后端负责好用
2. **可以换人**：换服务员不影响厨师，换厨师不影响服务员
3. **手机电脑都能用**：同一个厨房可以给不同餐厅供餐

## 二、后端技术（服务器端）

### 2.1 Java - 编程语言

**是什么？**
- 就像人类的语言有中文、英文，计算机也有很多编程语言
- Java 是最流行的编程语言之一，很多企业都用它

**为什么要用 Java？**
1. **工作好找**：国内大部分企业都用 Java
2. **资料多**：遇到问题网上都能找到答案
3. **稳定可靠**：银行、电信等重要系统都用 Java

**简单类比**：
- Java 就像普通话，虽然学起来有点难，但是全国通用

### 2.2 Spring Boot - 开发框架

**是什么？**
- 框架 = 别人写好的代码模板，你可以直接套用
- Spring Boot 是 Java 最流行的框架

**通俗理解**：
- 想象你要盖房子
- 不用框架 = 从烧砖、和水泥开始，什么都自己做
- 用 Spring Boot = 直接买预制板，拼装起来就行

**Spring Boot 帮我们做了什么？**
1. **自动配置**：不用写很多配置文件
2. **内嵌服务器**：不用单独安装 Tomcat
3. **自动管理依赖**：需要用什么工具，告诉它就行

**核心概念**：

#### ① Controller（控制器）
```
用户请求 ──► Controller ──► Service ──► Mapper ──► 数据库
                │
                ▼
            返回结果
```
- **作用**：接收用户的请求，决定怎么处理
- **类比**：餐厅的前台，接待客人，安排座位

#### ② Service（服务层）
- **作用**：处理业务逻辑
- **类比**：餐厅的厨师，真正做菜的人
- **例子**：
  - 计算考试成绩
  - 自动组卷算法
  - 判断用户是否有权限

#### ③ Mapper/DAO（数据访问层）
- **作用**：和数据库打交道，存取数据
- **类比**：餐厅的仓库管理员，负责取食材、存食材

#### ④ Entity（实体类）
- **作用**：定义数据的结构
- **类比**：菜单上的菜品描述（名称、价格、配料）

### 2.3 MyBatis-Plus - 数据库操作工具

**是什么？**
- 一个简化数据库操作的工具
- 不用写 SQL 语句，用 Java 代码就能操作数据库

**通俗理解**：
- 不用 MyBatis-Plus = 用英语给外国厨师写菜单
- 用 MyBatis-Plus = 直接用手指图片点菜

**核心功能**：

#### ① 自动生成 SQL
```java
// 原来要写这么多 SQL
SELECT * FROM user WHERE id = 1;

// 现在只需要
userMapper.selectById(1);
```

#### ② 代码生成器
- 自动生成 Entity、Mapper、Service、Controller
- 节省 80% 的重复代码

#### ③ 分页查询
```java
// 一行代码实现分页
Page<User> page = userMapper.selectPage(new Page<>(1, 10), null);
```

### 2.4 JWT - 用户身份认证

**是什么？**
- JWT (JSON Web Token) = 用户的"身份证"
- 用户登录后，服务器发一个"令牌"，之后用户带着这个令牌访问

**通俗理解**：
- 想象你去健身房
- 第一次：办会员卡（登录），给你一张卡（Token）
- 之后：每次去出示会员卡，就能进去

**为什么要用 JWT？**
1. **无状态**：服务器不用记住每个用户，减轻压力
2. **安全**：令牌有签名，防止伪造
3. **跨平台**：手机、电脑、小程序都能用

**工作流程**：
```
1. 用户登录 ──► 2. 服务器验证密码 ──► 3. 生成 JWT Token
                                              │
4. 用户访问其他页面 ◄── 5. 验证 Token ◄──┘
```

### 2.5 Maven - 项目管理工具

**是什么？**
- 管理项目依赖的工具
- 就像手机的"应用商店"，需要什么功能就去下载

**pom.xml 文件**：
```xml
<!-- 告诉 Maven：我需要这些工具 -->
<dependencies>
    <!-- Spring Boot -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    
    <!-- MySQL 连接 -->
    <dependency>
        <groupId>mysql</groupId>
        <artifactId>mysql-connector-java</artifactId>
    </dependency>
</dependencies>
```

## 三、前端技术（用户看到的界面）

### 3.1 Vue 3 - 前端框架

**是什么？**
- 用来制作网页的框架
- 让网页可以像 App 一样流畅

**通俗理解**：
- 不用框架 = 用记事本写网页，每个页面都要重新写
- 用 Vue 3 = 用乐高积木搭网页，可以复用组件

**核心概念**：

#### ① 组件化开发
```
页面 = 组件A + 组件B + 组件C

例如：
考试页面 = 头部导航 + 题目列表 + 答题卡 + 倒计时
```
- **好处**：组件可以重复使用，改一处到处生效

#### ② 数据双向绑定
```
数据变化 ──► 页面自动更新
页面输入 ──► 数据自动更新
```
- **类比**：Excel 表格，改一个单元格，相关公式自动计算

#### ③ 响应式系统
- 数据变了，页面自动刷新
- 不用手动操作 DOM（网页元素）

### 3.2 TypeScript - 带类型的 JavaScript

**是什么？**
- JavaScript 的超集，加了"类型检查"
- JavaScript 就像宽松的老师，TypeScript 就像严格的老师

**通俗理解**：
```javascript
// JavaScript（容易出错）
let age = 20;
age = "二十岁";  // 不会报错，但逻辑错了

// TypeScript（提前发现错误）
let age: number = 20;
age = "二十岁";  // 报错！类型不匹配
```

**为什么要用 TypeScript？**
1. **提前发现错误**：写代码时就知道哪里错了
2. **代码提示**：IDE 会自动提示有哪些方法
3. **易于维护**：大型项目必备

### 3.3 Element Plus - UI 组件库

**是什么？**
- 现成的网页组件（按钮、表格、表单等）
- 直接拿来用，不用自己设计样式

**提供的组件**：
- **基础组件**：按钮、图标、文字链接
- **表单组件**：输入框、下拉选择、单选框、多选框
- **数据展示**：表格、分页、标签、进度条
- **导航组件**：菜单、面包屑、标签页
- **反馈组件**：对话框、消息提示、加载动画

**使用示例**：
```vue
<!-- 一个漂亮的按钮 -->
<el-button type="primary">主要按钮</el-button>

<!-- 一个带分页的表格 -->
<el-table :data="tableData">
  <el-table-column prop="name" label="姓名" />
</el-table>
<el-pagination :total="100" />
```

### 3.4 Pinia - 状态管理

**是什么？**
- 管理整个应用的数据（状态）
- 就像一个"全局变量仓库"

**为什么要用？**
```
场景：用户登录后，多个页面都要显示用户名

不用 Pinia：
- 每个页面都要重新获取用户信息
- 数据不同步，一个页面改了，其他页面不知道

用 Pinia：
- 登录后存到"仓库"
- 所有页面都从仓库取数据
- 仓库数据变了，所有页面自动更新
```

**核心概念**：
- **State**：存储的数据（用户信息、考试状态等）
- **Getters**：计算属性（根据数据计算出的值）
- **Actions**：操作方法（修改数据的函数）

### 3.5 Axios - HTTP 请求库

**是什么？**
- 用来发送网络请求的工具
- 前端和后端交流的"信使"

**通俗理解**：
- Axios = 快递员
- 前端 = 发件人
- 后端 = 收件人
- API 接口 = 收件地址

**使用示例**：
```javascript
// GET 请求 - 获取数据
axios.get('/api/users')
  .then(response => {
    console.log(response.data);
  });

// POST 请求 - 提交数据
axios.post('/api/login', {
  username: 'admin',
  password: '123456'
});
```

### 3.6 Vite - 构建工具

**是什么？**
- 把源代码转换成浏览器能运行的代码
- 开发时的"翻译官"和"加速器"

**功能**：
1. **快速启动**：秒开开发服务器
2. **热更新**：改代码后页面自动刷新
3. **打包优化**：发布时压缩代码，提高性能

**类比**：
- 开发时 = 实时翻译，边说边译
- 打包时 = 提前翻译好，装订成书

## 四、数据库（存数据的地方）

### 4.1 MySQL - 关系型数据库

**是什么？**
- 存储数据的软件
- 数据以"表格"的形式存储

**通俗理解**：
- 数据库 = 图书馆
- 表 = 书架
- 数据 = 书
- SQL = 找书的方法

**核心概念**：

#### ① 表（Table）
```
┌─────────────────────────────────────┐
│            users（用户表）            │
├────┬──────────┬──────────┬──────────┤
│ id │ username │ password │  role    │
├────┼──────────┼──────────┼──────────┤
│ 1  │  admin   │  ******  │  ADMIN   │
│ 2  │  teacher │  ******  │  TEACHER │
│ 3  │  student │  ******  │  STUDENT │
└────┴──────────┴──────────┴──────────┘
```

#### ② 字段（Column）
- 表的列，定义数据的属性
- 如：id、username、password

#### ③ 记录（Row）
- 表的行，一条数据
- 如：id=1 的用户信息

### 4.2 SQL - 数据库语言

**是什么？**
- 操作数据库的语言
- 增删改查数据

**常用语句**：
```sql
-- 查询（查）
SELECT * FROM users WHERE id = 1;

-- 插入（增）
INSERT INTO users (username, password) VALUES ('test', '123456');

-- 更新（改）
UPDATE users SET password = 'newpass' WHERE id = 1;

-- 删除（删）
DELETE FROM users WHERE id = 1;
```

### 4.3 Redis - 缓存数据库

**是什么？**
- 存在内存里的数据库
- 速度比 MySQL 快 10-100 倍

**通俗理解**：
- MySQL = 图书馆的书架（存得多，找得慢）
- Redis = 办公桌（存得少，拿得快）

**用途**：
1. **缓存**：把常用数据放内存，减少数据库压力
2. **会话存储**：存用户登录状态
3. **计数器**：统计在线人数、点赞数

## 五、开发工具和环境

### 5.1 Git - 版本控制

**是什么？**
- 记录代码修改历史的工具
- 可以回到之前的版本
- 多人协作不冲突

**核心概念**：

#### ① 仓库（Repository）
- 存代码的地方
- 本地仓库 = 你电脑上的代码
- 远程仓库 = GitHub 上的代码

#### ② 提交（Commit）
```
修改代码 ──► 添加到暂存区 ──► 提交（保存快照）
```

#### ③ 推送（Push）和拉取（Pull）
```
本地提交 ──► 推送 ──► 远程仓库（GitHub）
远程仓库 ──► 拉取 ──► 本地更新
```

**常用命令**：
```bash
# 查看状态
git status

# 添加文件
git add .

# 提交
git commit -m "修改了登录功能"

# 推送
git push origin main

# 拉取
git pull origin main
```

### 5.2 GitHub - 代码托管平台

**是什么？**
- 存放代码的"网盘"
- 全世界程序员都在用

**功能**：
1. **代码托管**：代码存云端，不怕丢失
2. **版本管理**：可以看历史修改
3. **协作开发**：多人一起写代码
4. **项目管理**：Issue、Project 等功能

### 5.3 IDE - 集成开发环境

**是什么？**
- 写代码的软件
- 提供代码提示、调试等功能

**常用 IDE**：
- **后端**：IntelliJ IDEA（Java 开发神器）
- **前端**：VS Code（免费、插件多）

```
                              │
                              ▼

                              │
                              ▼
┌────────────────────────────────────────────────────────────────┐
│  4. 生成试卷对象                                                 │

                              │
                              ▼
┌────────────────────────────────────────────────────────────────┐
│  5. 保存到数据库                                                 │
└────────────────────────────────────────────────────────────────┘
```

## 八、总结

### 技术栈总览

| 层级 | 技术 | 作用 |
|------|------|------|
| **前端** | Vue 3 | 构建用户界面 |
| | TypeScript | 类型安全的 JavaScript |
| | Element Plus | UI 组件库 |
| | Pinia | 状态管理 |
| | Axios | HTTP 请求 |
| | Vite | 构建工具 |
| **后端** | Java | 编程语言 |
| | Spring Boot | 开发框架 |
| | MyBatis-Plus | 数据库操作 |
| | JWT | 身份认证 |
| | Maven | 项目管理 |
| **数据库** | MySQL | 关系型数据库 |
| | Redis | 缓存数据库 |
| **工具** | Git | 版本控制 |
| | GitHub | 代码托管 |

### 学习建议

1. **先学基础**：HTML/CSS/JavaScript → Java 基础
2. **再学框架**：Vue 3 → Spring Boot
3. **边做边学**：跟着项目代码看，不懂就查
4. **多查资料**：MDN、Vue 官网、Spring 官方文档

### 答辩时可以说

- "本项目采用前后端分离架构，前端使用 Vue 3 + TypeScript，后端使用 Spring Boot"
- "数据库使用 MySQL 存储业务数据，Redis 做缓存"
- "使用 JWT 实现无状态的身份认证"
- "自动组卷功能采用随机抽取算法，支持按题型、难度、学科筛选"
