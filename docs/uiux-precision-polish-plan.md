# UI/UX 精工打磨规范（PC端）

## 1. 目标与边界

### 1.1 目标
- 在不改变业务功能的前提下，提升界面的“精工感”：统一、克制、可预期、低摩擦。
- 优先打磨交互细节与视觉节奏，不追求“花哨”，追求“稳定且高级”。
- 同步提升可维护性：减少重复样式、统一组件行为、降低页面间风格漂移。

### 1.2 明确边界（本轮必须遵守）
- 不新增业务功能、不删除现有功能、不改权限逻辑。
- 仅做 PC 端体验优化，不新增响应式功能需求。
- 数据结构仅限已确认范围（课程封面字段已落地），不扩展新业务字段。

## 2. 当前项目整体观察（前端）

### 2.1 已有优势
- 已建立设计令牌体系（`frontend/src/styles/design-tokens.scss`）。
- 列表页有基础复用层（`base-list.scss` / `base-detail.scss`）。
- 多数页面已采用黑白灰主视觉，方向正确。

### 2.2 主要问题
- **交互反馈不一致**：按钮、卡片、表格、标签页的 hover/focus 细节不统一。
- **视觉密度不稳定**：部分页面标题、卡片、表格行高与间距节奏存在偏差。
- **组件复用不足**：局部样式（如操作按钮、卡片点击态）有重复实现。
- **头像/封面链路历史不完整**：顶部头像未绑定 src（已修复），课程封面刚接入需统一表现。

## 3. 精工标准（执行基线）

### 3.1 视觉公差（统一标准）
- 圆角：主卡片 `6px`，小元素 `4px`。
- 边框：主边框统一 `1px`，颜色统一 token。
- 间距：仅使用 4/8/16/24/32 体系，禁止随意“魔法数”。
- 字重：正文 `400`，强调 `500/600`，避免同页多个不必要字重层级。

### 3.2 交互标准
- 所有可操作元素必须有 hover + focus-visible 反馈。
- 动画统一短时长（100–180ms），避免突兀或拖沓。
- 可点击卡片必须有一致的“可点击信号”（边框/背景/微位移中的一种或组合）。

### 3.3 信息呈现标准
- 封面图比例统一（课程卡片统一比例与占位）。
- 表格头部与正文行高统一，避免“忽胖忽瘦”。
- 空状态文案简洁，避免情绪化或冗余表达。

### 3.4 代码质量标准
- 同类样式尽量回收至全局或基础层，减少页面私有重复定义。
- 保持 token 驱动，不写散落的硬编码颜色/阴影。
- 组件 props/emits 使用类型约束，减少隐式行为。

## 4. 执行顺序（本轮）

### Phase A：基础层（全局）
1. 统一交互反馈（focus-visible、hover 过渡、按钮/输入微动效）。
2. 统一标题与容器节奏（page-header、table-card、dialog 细节）。
3. 补齐全局可点击元素的键盘可达反馈。

### Phase B：关键页（高频）
1. 主框架区（顶部用户区、侧栏选中态、面包屑精度）。
2. 首页（学生/教师）卡片与封面风格统一。
3. 课程页（列表/详情）封面、标签、信息层级统一。

### Phase C：质量收口
1. 编译校验（frontend build、backend compile）。
2. 回归检查（不变更业务路径）。
3. 输出“完成项 + 下一轮候选项”。
