<!-- _header: '1. 课题背景与研究意义' -->
## 1.1 课题背景

随着信息技术的飞速发展和“互联网+教育”战略的持续推进，教育领域的数字化转型已成为不可逆转的趋势。

- **传统考试模式的局限与痛点**：传统纸质考试依赖人工出题、印刷、监考和人工阅卷。整个周期长，工作量巨大，且极易出现人为统分错误，保密成本高昂。
- **在线教育的常态化普及**：近年来，打破时间与空间限制的线上教学模式被广泛接受，与之配套的考核环节也亟需从“线下”向“线上”迁移。
- **现有个性化需求的增长**：不同高校或培训机构针对教学管理有其特定的流程，急需一款轻量、易于二次开发与定制的考试平台，以取代笨重且昂贵的商业软件。

---

<!-- _header: '1. 课题背景与研究意义' -->
## 1.2 研究意义

开发一套高效、稳定的基于 Java 的在线考试系统，具有重要的理论与现实意义：

1. **提升教学与教务效率**：
   - 彻底解放教职工在传统阅卷、统分中的繁重劳力。
   - 实现题库的电子化沉淀，方便题目的检索、修改与复用。
2. **多维度的教学数据支撑**：
   - 系统能够直观呈现各科目的考试成绩分布规律、知识点得分率分析。
   - 为教师调整后续教学方案提供真实、客观的数据支撑。
3. **促进绿色低碳发展**：
   - 真正实现“无纸化”办公与测试，大幅度削减纸张及印刷耗材成本，符合环保理念。

---

<!-- _header: '2. 国内外研究现状' -->
## 2.1 国内研究现状

国内的在线考试平台发展极为迅速，已经形成了多种形态的市场格局：

- **大型商业级平台**（如：超星学习通、雨课堂、问卷星、腾讯课堂）：
  - **优势**：功能大而全，并发承载能力强，生态完善。
  - **不足**：作为公有云 SaaS 服务，难以满足部分学校敏感数据的私有化部署需求；且部分高级功能收费昂贵。
- **高校自研教务考试系统**：
  - **优势**：与校内学工系统和人事系统集成度高。
  - **不足**：大多采用早期的 JSP 或 Struts 架构，界面陈旧，用户交互体验较差，移动端适配不足，系统维护成本极高。

---

<!-- _header: '2. 国内外研究现状' -->
## 2.2 国外研究现状及综合评述

**国外研究现状**：
- 国外普遍以综合性的 **LMS（学习管理系统）** 为主导，例如著名的开源系统 **Moodle**、Canvas 以及商业系统 Blackboard。
- 这些系统通常包含丰富的插件生态，考试只是其中的一个模块。但其系统过于庞杂，部署与二次开发门槛较高，对于单纯只需要“在线测试”功能的场景显得过于重度。

**综合评述与本课题的切入点**：
- 综上所述，目前市场和开源社区 <span class="highlight">缺乏一款纯粹、轻量、现代化前后端分离技术栈栈（如 Spring Boot + Vue）的优质在线考试系统</span>。
- 本课题旨在填补这一空白，完成一套轻便、私有化部署友好、代码结构清晰的考试系统，以满足中型规模的机构与院校日常测评需求。

<!-- _header: '3. 研究目标与主要内容' -->
## 3.1 研究目标

本项目旨在设计并实现一套架构合理、性能良好、界面友好的在线考试系统。

**核心设计目标：**
- **角色清晰**的权限控制：系统需具备管理员、教师和学生三类角色独立的视图与操作权限。
- **高可用性设计**：保证在多数用户同时登录并提交试卷时，系统能平稳运行，不出现丢失数据的异常。
- **良好的用户体验**：前端界面需遵循现代 UI 设计规范，操作逻辑简单易懂。

---

<!-- _header: '3. 研究目标与主要内容' -->
## 3.2 系统的主要功能划分 (1/2)

系统主要划分三大功能模块：**学生端**、**教师端**与**管理员端**。

**学生端（前台用户交互）**：
- **个人中心**：资料修改、密码重置。
- **在线考试模块**：浏览当前可考科目，进入考试、倒计时提醒、答题界面切换、自动保存草稿、防切屏警告与强制交卷。
- **成绩查询模块**：查看过往考试历史得分，及主观题教师评语，查阅客观题的正确答案及解析。

---

<!-- _header: '3. 研究目标与主要内容' -->
## 3.2 系统的主要功能划分 (2/2)

**教师端（核心业务处理）**：
- **题库管理**：题目（单选、多选、判断、简答）的增删改查、Excel 批量导入。
- **试卷管理**：人工从题库挑选题目组卷，或设定知识点与难度比例进行系统随机抽题组卷。
- **考试发布与监控**：设定考试开放时间、时长，查看当前考试的实时答题进度。
- **阅卷与统计**：对未支持自动判分的简答题进行人工阅卷打分，系统最终汇总总分并生成班级成绩报表（均分、及格率等图表）。

**管理员端（系统基础维护）**：
- **用户管理**：学生、教师账号的批量创建与状态控制。
- **系统设置**：角色权限菜单分配、字典表维护等系统级配置。

<!-- _header: '4. 系统方案与关键技术' -->
## 4.1 后端技术选型

系统后端主要采用 Java 企业级开发生态，保证了系统的跨平台性和可维护性。

- **核心框架**：`Spring Boot`（版本可根据实际选型，如 2.7 或 3.x）。它规避了传统 Spring 繁琐的 XML 配置，提供开箱即用的特性。
- **持久层框架**：`MyBatis-Plus`。在 MyBatis 基础上增加强大的 CRUD 操作支持，极大减少基础 SQL 的编写工作量。
- **安全与权限框架**：`Spring Security` 或自定义 `JwtInterceptor` 加 `注解`拦截。通过引入 `JWT (JSON Web Token)` 实现无状态登录，不仅提升了访问性能，也完美契合前后端分离架构。
- **数据库连接池与工具**：使用 `Druid` 连接池管理数据库连接，使用 `Lombok` 简化实体类开发。

---

<!-- _header: '4. 系统方案与关键技术' -->
## 4.2 前端及存储技术选型

**前端技术栈**：
- **基础框架**：`Vue.js`（Vue2 / Vue3）构建渐进式单页应用（SPA）。
- **路由与状态管理**：`Vue Router` + `Pinia/Vuex`。
- **UI 组件库**：`Element UI / Element Plus`，提供美观、统一的后台与前台交互组件（如表格、表单、对话框）。
- **扩展组件**：网络请求封装 `Axios`；图表可视化展示使用 `ECharts`。

**数据存储技术栈**：
- **主数据库**：`MySQL`（版本 5.7/8.0），利用其关系型特性存储复杂的题库、考试关联模型。
- **缓存机制（备选进阶）**：考虑引入 `Redis`，对考试过程中的临时答题记录、热门考卷信息以及 Token 生命周期进行缓存优化，缓解直接打库的压力。

---

<!-- _header: '5. 系统架构与核心难点' -->
## 5.1 系统架构设计 (B/S 与前后端分离)

本项目严格采用 **B/S (Browser/Server) 架构** 和 **前后端分离** 模式。

- 前端（Client）与后端（Server）在物理与逻辑上完全解耦，二者通过标准的 `RESTful API (JSON)` 进行数据交互。
- 这种设计的优势在于：
  1. 提升了开发效率（前后端可并行开发验证）。
  2. 减轻了服务器端渲染页面的压力，界面跳转更加流畅。
  3. 后续若需要开发移动端（如微信小程序），只需复用现有后端 API。

---

<!-- _header: '5. 系统架构与核心难点' -->
## 5.2 核心难点与应对策略

在本系统开发过程中，预计会面临以下几个技术与逻辑层面的难点：

1. **考试防作弊机制的实现**：
   - *策略*：前端通过监听浏览器 `visibilitychange` 事件和 `window.onblur` 焦点事件，记录学生切屏次数。如果切屏超过设定阈值，系统触发自动交卷。
2. **答题异常中断与状态保持**：
   - *策略*：为了防止学生考试中因断电断网丢失答题记录，前端使用浏览器缓存储存考卷草稿，且定时向后端异步提交心跳保存记录。
3. **随机组卷的算法优化**：
   - *策略*：在满足特定难度系数、特定知识点权重的前提下，优化题库查询算法并使用事务保证组卷过程中数据的一致性。
